# Auto-Save 통일 설계

## 현황

프론트엔드에 5개의 서로 다른 자동 저장 구현이 존재한다.

| 위치 | 디바운스 | 방식 |
|------|----------|------|
| 워크플로우 캔버스 | 2초 | Zustand 스토어 구독 + 스냅샷 비교 |
| 세션 설정 뷰 | 2초 | 공통 훅 (`shared/hooks/useAutoSave`) |
| 파이프라인 설정 | 0.8초 | 인라인 useEffect + setTimeout |
| 편집 방향 설정 폼 | 1초 | 인라인 useEffect + setTimeout |
| 분석 결과 편집 | 1.5초 | React Query mutation + setTimeout |

공통 훅이 이미 잘 설계되어 있지만 세션 설정 뷰 한 곳에서만 사용 중이다.

## 목표

5곳 전부 `shared/hooks/useAutoSave` 훅을 직접 사용하도록 통일한다.

## 설계 결정

- **공통 훅 변경 없음**: 현재 훅이 이미 필요한 기능을 모두 갖추고 있음
- **래퍼 없음**: 캔버스 포함 모든 곳에서 공통 훅 직접 호출
- **디바운스 2초 통일**: 페이지 이탈 시 저장이 되므로 입력 중 빈번한 저장 불필요
- **저장 상태 5단계 통일**: 유휴 → 대기 → 저장 중 → 저장됨 → 에러

## 마이그레이션 상세

### 워크플로우 캔버스

현재 Zustand 스토어 구독으로 노드/연결선/이름 변경을 감지한다. 공통 훅으로 전환 시:

- 캔버스 상태(노드, 연결선, 이름)를 하나의 데이터 객체로 꺼내서 공통 훅에 전달
- 실행 중이거나 캔버스가 비었거나 템플릿이면 저장 비활성화
- 이름 없는 워크플로우의 이름 자동 생성은 저장 함수 안에서 처리
- 기존 반환 인터페이스(저장 상태, 즉시 저장, 초기화) 동일하게 유지 — 소비하는 쪽 변경 없음

### 파이프라인 설정

현재 소스, 스케줄, 워크플로우, 모델 4개를 각각 상태로 관리하고 직접 디바운스한다.

- 4개를 하나의 설정 객체로 묶어서 공통 훅에 전달
- 페이지 이탈/브라우저 닫기 시 저장을 공통 훅이 처리 (현재도 있지만 직접 구현)
- 기존 인라인 디바운스/dirty 감지/unmount 저장 코드 제거

### 편집 방향 설정 폼

현재 자동 저장 모드에서 직접 디바운스하고, 수동 저장도 별도 구현.

- 공통 훅 사용. 수동 저장은 훅의 즉시 저장 기능으로 대체
- 첫 렌더링 스킵 로직 불필요 — 공통 훅이 dirty 비교로 불필요한 저장 방지

### 분석 결과 편집

현재 React Query mutation으로 요약/인사이트 편집을 저장.

- 공통 훅 사용. 저장 함수 안에서 API 호출
- React Query mutation 제거, 공통 훅의 에러 처리로 통일

## 변경되지 않는 것

- 공통 훅 자체 (`shared/hooks/useAutoSave.ts`)
- 각 페이지의 UI/UX — 저장 타이밍(2초)과 상태 표시만 통일
- 백엔드 API 엔드포인트
